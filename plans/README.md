# Plans

功能级新增或修改前，必须先在本目录写实施计划。

## 文件命名

计划文件命名使用：

```text
序号-状态-计划总结.md
```

示例：

```text
001-draft-discord-alert-allowlist.md
002-approved-initial-schema.md
003-done-shadow-alert-projection.md
```

## 内容要求

每份计划至少说明：

- 目标和成功标准
- 涉及的配置、schema、接口或行为变化
- 实施步骤
- 测试与验收方式
- 明确不做的内容

typo、格式、纯注释等不改变行为的小修可以不写计划。
